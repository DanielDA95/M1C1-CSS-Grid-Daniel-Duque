Pregunta 1

¿Cuáles son algunas de las cosas que hacen que SCSS sea diferente de CSS?

Respuesta

El Scss es un archivo SASS y es un preprocesador de codigo que ayuda a organizar mejor el codigo en css y evita las codigos repetitivos en las hojas de css; el SCSS se procesa directamente en el servidor y genera un archivo css para que el navegador lo pueda entender. Ejemplo. En scss se pueden crear variables, que en css se podria repetir varias veces en la hoja de estilo, pero la variable ayuda para que no se repita este codigo.

Pregunta 2

¿Qué es una variable SCSS?

Respuesta

Una variable es un tipo de contenedor de codigo (informaicon), que se puede llamar desde cualquier parte del codigo en css para no reperir en ocaciones un mismo codigo en diferentes partes de la hoja de estilo, y por el contrario se use solo la variable, dando a entender que este es el codigo que necesita para ese estilo.
Ejemplo de una varialble y su sintaxys, primero se usa el signo $ y se da el nombre a la variable, luego se nombra valor que se va a asignar,
ej: $color-subtitulos : #282828;
En el anterior ejemplo de variable se asigno el valor para que todos los titulos que se pongan en la pagina tenga un color gris.

Pregunta 3

¿Qué es un SCSS Mixin?

Respuesta

Un mixin es un conjunto de estilos agrupados que al igual que las variables se puede llamar desde cualquier parte de la hoja de estilos y nos permite tener un comportamiento más dinamico y tambien tener unos estilos predeterminados para secciones de codigo que por ejemplo, necesiten un margen, tamaño de fuente, tipo de fuente y color iguales. Esto nos ahorra esa repetición de codigo en la hoja de estilo. La sintaxis de un mixin tambien puede ser predeterminada.
Ejemplo:
@mixin contenido-preguntas-respuestas() {
$color-subtitulos;
background-color: Blue;
color: black;
margin-top: 25px;
}

Pregunta 4

¿Qué significa Unidad fraccionaria (fr) con CSS Grid?

Respuesta

La unidad fraccionaria es una forma de medicion en css, que permite dividir una cuadricula en columas o filas proporcionalmente, sin tener que recalcular el ancho o alto de estos elementos.
